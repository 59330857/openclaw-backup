## 为什么在 Dashboard 只看到主 Agent？

1. **会话类型区分**  
   - **主会话** 是用户直接交互的会话（即当前对话的 `sessionKey`）。  
   - **子 Agent** 是通过 `sessions_spawn` 创建的 **孤立子会话**（`mode="run"` 或 `mode="session"`），它们拥有独立的 `sessionKey`，但默认 **不在主会话列表中露出**，除非主会话主动 **公布** 或 **引用** 它们。

2. **默认 UI 过滤**  
   - Dashboard 的会话列表只显示 **`kind=systemEvent`** 或 **`kind=agentTurn`** 且 **`sessionTarget="main"`** 的会话。子 Agent 的 `sessionTarget` 被设为 `isolated`（或 `current`），因此不会出现在主会话列表。  
   - 只有当子 Agent **主动发送消息回主会话**（例如通过 `sessions_send` 把结果回传），或者你使用特定工具查询子会话的信息时，才会在 UI 中出现。

3. **安全/信息分离**  
   - 子 Agent 往往承担独立任务（如本例的 **TikTok 抓取、评分、ROI 计算**），为避免信息泄露，系统默认不把它们直接列入主会话页面。只有在需要人工审查或后续交互时，才会通过 **子会话 ID** 把结果回传或查询。

## 常见原因导致子 Agent 不在 Dashboard 显示

| 原因 | 说明 | 解决办法 |
|------|------|----------|
| **未公开子会话** | 子 Agent 使用了 `sessionTarget="isolated"`，默认不向主会话抛出公告。 | 在子 Agent 的 **`payload.delivery.mode`** 设为 `"announce"` 或在完成后手动 `sessions_send` 把信息回传。 |
| **会话列表过滤** | Dashboard 默认过滤掉 `sessionTarget!="main"` 的会话。 | 使用 `openclaw sessions list --all` 或在 UI 右上角切换 “显示全部会话”。 |
| **子 Agent 还在启动中** | 新创建的子 Agent 需要几秒钟完成初始化（网络、依赖安装）。 | 稍等 10–30 秒后再次执行 `openclaw sessions list --all`。 |
| **权限不足** | 查看子会话需要 **系统管理员** 或 **高级功能** 权限。 | 确认你的用户已加入 `openclaw-admin` 组，或执行 `openclaw role grant <your_user> admin`。 |
