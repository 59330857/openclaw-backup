## 零、入场时间窗口（跟品核心过滤器）—— ★ v5.2 新增核心

> 所有轨道的**统一前置标准**，不符合的品直接淘汰，不进入后续评分流程
> 核心逻辑：卡住入场时间节点，找刚启动爆发的品，避免跟盘已饱和的品

### 跟品入场七项过滤指标

| 指标 | 通过标准 | 淘汰逻辑 |
|------|---------|---------|
| **上架时间** | ≤30 天 | 超过30天的品，竞争对手已进场，无先发优势 |
| **近7天销量增长率** | ≥50% | 说明算法正在推流，产品需求处于爆发期 |
| **总销量** | 100-500 单 | 太少未经过市场验证，太多说明已爆，无红利空间 |
| **评价数** | 50-200 条 | 有真实用户反馈，可确认产品无硬伤 |
| **同款卖家数量** | ≤20 个 | 超过20个会启动价格战，压缩利润空间 |
| **客单价** | 3-10 美元 | 与当前爆款客单价一致，用户画像完全匹配，转化更顺畅 |
| **退货率** | ≤10% | 发饰类正常退货率，超过说明产品存在质量或适配问题 |

### 与SOP三轨的关系

| 关系 | 说明 |
|------|------|
| **不是单独轨道** | 是所有轨道的统一前置过滤器 |
| **先于参数卡死** | 先过滤入场时间窗口，再做静态参数卡死 |
| **适配跟品逻辑** | 爆款=已验证，无需复杂三步验证；入场时机才是关键 |

### 判断结论

```
七项全部通过 → 进入三轨选品流程
任一项不通过 → 直接淘汰，不进入后续流程
```

## 一、参数卡死（源头过滤红海）—— 亚妈逊精铺精髓

亚妈逊精铺第一步：用固定参数筛红海，从源头过滤大部分无效品
**TK版本完全适用，参数需根据TK市场特点调整**

### 参数设置（直接抄作业）

| 参数 | 保守标准 | 常规标准 | 目的 |
|------|---------|---------|------|
| **月销量** | ≤200 单/月 | ≤500 单/月 | 避免头部扎堆爆品赛道 |
| **评分数**（匹配上架时间） | 近半年上架 ≤30 条 | 近两年上架 ≤100 条 | 低评分也能稳转化，运营压力小 |
| **客单价** | ≥30 美金等效 | ≥20 美金等效 | 过滤低价内卷、利薄盖不住广告的品 |
| **配达方式** | FBA+FBM 全渠道 | FBM 为主 | 全渠道竞品不遗漏（TK以本地发货为主） |
| **季节匹配** | 未来 3-4 个月为应季 | 未来 2-3 个月为应季 | **时光机法则：提前布局，接住流量爆发** |

### 时光机法则（关键！亚妈逊精髓）

```
4月选 8月应季 → 货刚好赶上流量爆发
4月选 9月应季 → 国庆节/圣诞提前备货
```

### 参数卡死落地工具

- **亚妈逊：** 卖家精灵「选产品」→ 直接照抄参数
- **TK版本：** EchoTik 筛选器 + 手动参数核对
  - 月销量 → EchoTik 商品列表「月销量」排序
  - 评分数 → 商品详情页「评论数」
  - 上架时间 → 商品详情页「上架时间」

## 二、三步快速验证（亚妈逊精铺核心 → TK适配版）—— ★ v5.0 新增核心

> **亚妈逊方法论：** 精准词验证可行性 → 竞品算成本 → 保守测盈利
> **核心逻辑：** 在深入分析前，用最少步骤判断这个品值不值得做

### 第一步：查精准属性词（判断转化潜力）

**亚妈逊逻辑：** 找销量最好的直接竞品 → H10 反击关键词 → 无精准词直接 PASS
**TK版本：**

```
步骤：
1. 在 EchoTik 找到目标品类销量最好的直接竞品
2. 反击其关联的 Hashtag 标签 / 达人次标签
3. 判断：有精准标签 → 好转化、易运营；无精准标签 → 直接 PASS

判断标准：
- 有 3+ 个精准标签（越南语/泰语）→ 转化有保障
- 精准标签与目标人群匹配 → 内容种草友好
- 无精准标签或标签模糊 → 市场教育成本高，慎入
```

### 第二步：查关键词实时竞品（算广告/流量成本）

**亚妈逊逻辑：** 用无出单记录的低权重竞品反击，数据更保命
**TK版本：**

```
步骤：
1. 找到低权重竞品（0基础视濒、无粉丝账号也能出单的品）
2. 反击其短视频 CPM / 达人佣金比例
3. 估计流量成本对盈利底线的影响

关键指标：
- 短视频 CPM（千次播放成本）：判断内容投放性价比
- 达人佣金比例：10-20% 正常，超过 25% 压缩利润
- 自然流量占比：自然流量占比高 → 广告依赖低 → 更健康
```

### 第三步：保守测盈利（决定最终是否深 入）

**亚妈逊逻辑：** 假设转化率 5%-7%、无自然流量 → 用选品利润表计算
**TK版本（完全沿用）：**

```
假设条件（保守）：
- 转化率：5%-7%（参考同类爆款）
- 自然流量：暂不计，自然流量是额外安全垫
- 达人佣金：10%（可上下浮动）

计算公式：
真实毛利润 = 售价_RMB - 进货价 - 物流费 - 总平台成本（41%+¥2）
真实毛利润率 = 真实毛利润 / 售价_RMB

结论判断：
- 毛利润率 ≥ 30%（单品）/ ≥ 45%（套装）→ 盈利，继续做竞品分析
- 毛利润率 20-30% → 微利，需结合其他维度综合判断
- 毛利润率 < 20% → 直接放逐
```

## 八、备货黄金法则（亚妈逊精铺 → TK适配）—— ★ v5.0 新增核心

> 亚妈逊：市场月体量≈1000单、头部月销≈200单 → 首单80-100件，目标月销50单
> **TK逻辑：体量更小、更分散，首单应更保守**

### TK发饰备货法则

```
判断基准：
- 市场月体量 ≈ 500-800 单（越南）/ 200-400 单（泰国）
- 头部月销 ≈ 100-200 单（越南）/ 50-100 单（泰国）

首单计算（保守法则）：
首单备货量 = EchoTik 热卖款月销量 × 20-30%
上限：不超过 100 件
下限：不低于 30 件（满足变体测试需求）

目标月销：30-50 单（越南）/ 20-30 单（泰国）
```

### 备货决策矩阵

| 市场信心 | 首单策略 |
|---------|---------|
| 热卖款月销 100-200 单 | 首单 30-50 件，测试 2 周 |
| 热卖款月销 200-500 单 | 首单 50-80 件，测试 3 周 |
| 热卖款月销 500+ 单 | 首单 80-100 件，测试 4 周 |
| 新品/无数据 | 首单 30 件，快速测款 |

### 补货法则

```
2周内动销 ≥ 30% → 立即补货（追单）
2周内动销 15-30% → 正常补货
2周内动销 < 15% → 停止补货，下架处理
```

## 九、发货数量 + 变体循环布局

### 亚妈逊逻辑

- 首批发货：参考≥30评竞品平均销量的50%
- 变体布局：引流转体 → 销售变体 → 备用变体 → 补货变体
- 分散库存，防差评影响全链接触

### TK发饰版本（变体堆叠是杀手！）

| 变体维度 | 可扩展数量 | 效果 |
|---------|-----------|------|
| 颜色变体 | 5-10 色 | 评论共享，5个颜色=1个链接 |
| 规格变体 | 3+ 种 | S/M/L 多规格覆盖 |
| 套装形式 | 2+ 种 | 2件套/5件套/节日礼盒 |

### 变体堆叠落地策略

```
一次性上架所有颜色变体 → 评论全部累计共享
↓
5个颜色只有1个链接 → 评价积累速度 x5
↓
搜索权重集中 → 自然流量集中
↓
降低广告依赖 → 利润率提升
```

### 变体布局时序

1. **测试变体：** 先上3-5个颜色测试
2. **爆量变体加码：** 哪个颜色出单快，快速补充库存
3. **滞销变体淘汰：** 2周无动销则下架
4. **节日变体前置：** 节日45天前准备好特殊颜色/包装

## 十、评估新品短视频种草友好度（新品好不好起）

### 亚妈逊逻辑

看新品（≤30评）前5销量 ÷ 头部前5销量 > 20%

### TK版本：看短视频冷启动能力

| 信号 | 含义 |
|------|------|
| **0基础视濒也能跑出量** | 市场需求肥，内容驱动有效 |
| **无粉丝账号发视濒也能出单** | 对新手友好，流量分配去中心化 |
| **低评分也能卖** | 市场不饥和，评分体系尚未固化 |
| **FBM/Shipping也稳定出单** | 物流不是核心瓶颈 |

### TK核心冷启动指标

- **新品视濒发出48h内自然播放量**
- **0基础账号发视濒的带贷转化率**
- **同类视濒平均GMV vs 新品GMV占比**

### 判断标准

```
新品短视频种草系数 = 新品48h播放量 / 同类平均播放量
> 30% = 对新品友好，市场机会大
< 15% = 新品难以起量，慎入
```

## 选品 SOP 完整流程（v5.2 新增入场过滤）

```
第零步：入场时间窗口（5分钟）★ 新增前置
├─ 上架时间 ≤30天 ✓
├─ 近7天销量增长率 ≥50% ✓
├─ 总销量 100-500单 ✓
├─ 评价数 50-200条 ✓
├─ 同款卖家数量 ≤20个 ✓
├─ 客单价 3-10美元 ✓
└─ 退货率 ≤10% ✓
→ 不通过 → 直接淘汰
→ 全部通过 → 进入第一步

第一步：参数卡死（5分钟）
├─ 月销量 ≤500（越南）/ ≤300（泰国）
├─ 评分数 ≤100（近两年上架）
├─ 客单价 ≥20 美金等效
└─ 季节匹配：未来3-4个月为应季

第二步：三步快速验证（10分钟）
├─ 精准词验证：有 3+ 个精准标签 ✓
├─ 竞品/CPM合理：佣金 ≤25%，CPM 可接受 ✓
└─ 保守测盈利：毛利润率 ≥25%（单品）/ ≥45%（套装）✓

第三步：通用词延展（5分钟）
├─ 按场景/节日/人群延展 2-3 个细分变体
└─ 验证延展款销量是否达标

第四步：差异化 + 供应链确认（5分钟）
├─ 变体扩展性：颜色≥5 或 规格≥3
├─ 供应商：评分≥4.5，发货≤48h
└─ 合规检查：无 IP 侵权

第五步：评分 + 决策（5分钟）
├─ 8维度评分 ≥100 分
├─ 毛利达标
└─ 三步验证全部通过
→ 通过 → 写入飞书 Bitable → 采购征集
→ 不通过 → 记录放逐原因 → 下一款
```

## 行为准则

1. **数据真实性：** 绝不伪造数据，暂时无法获取的指标标记为"暂无数据"
2. **IP零容忍：** 发现任何IP侵权风险商品，无论利润多高，一律排除
3. **盈利优先：** 毛利率不达标（泰<25%/越<20%）的商品一律不推荐
4. **及时更新：** 当商品出现差评率上升、竞争加剧等信号时，主动下调等级或移出推荐池
5. **中文输出：** 所有报告、通知、摘要均使用中文
6. **三步验证前置：** 未通过三步验证的商品不做深入分析
7. **保守测算是底线：** 假设无自然流量，盈利才能做，不盈利坚决不做
